# 基于 Beaver 协议

现代许多协议基于 Beavers 的电路随机化概念。在这里，MPC 协议被分成两个阶段；一个离线阶段，产生所谓的乘法三元组，和一个在线阶段，用于实际函数的评估。如果我们用 [a] 表示价值 a 的秘密共享，那么离线阶段会产生这种形式的三元组([a]，[b]，[c])，其中 c = a\*b。离线阶段通常可以用于产生其他形式的相关随机性。

虽然离线阶段是函数和输入独立的，并且通常需要公钥操作，但在线阶段执行时函数和输入已知。该阶段会消耗 Beaver 三元组并评估函数。

采用一种认证秘密共享的形式获得主动安全，在完全阈值情况下，这是通过在每个秘密共享值上使用消息认证码的秘密共享来实现的。

根据所支持的底层算术类型和生成 Beaver 三元组的方式，这个家族有许多协议。

TINY-OT 最初是一个两方协议，该协议使用混淆传输来在离线阶段中产生三元组。Tiny-OT 协议对布尔值进行求值，是许多现代 BMR 协议的基础。

SPDZ 是任何有限域的 n 方全阈值协议。离线阶段通常基于同态加密，尽管基于 Tiny-OT 的变体也是可能的(在这种情况下，称该协议使用 MASCOT 作为离线阶段)。基于现代同态加密的离线阶段有三个变体 HighGear、LowGear 和 TopGear，取决于所使用的加密方案和零知识证明类型。

SPDZ2K 是适用于 Z\_{2^k}环上传输的 SPDZ 变体。离线阶段可以基于混淆传输或同态加密。
